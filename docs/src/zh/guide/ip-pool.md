# IP 池管理 {#ip-pool}

IP 池用于管理节点的可分配 IP 地址。每个节点可以创建多个 IP 池，系统会在创建实例时自动从对应节点的 IP 池中分配空闲 IP。

在创建套餐之前，您必须先为对应节点创建 IP 池并添加足够的 IP 地址，否则用户购买套餐时会因为无法分配 IP 而失败。

## 创建 IP 池 {#create}

在管理面板的「IP 池管理」页面，点击「创建 IP 池」，填写以下信息：

| 字段 | 说明 |
|------|------|
| 名称 | IP 池的显示名称 |
| 所属节点 | 该 IP 池关联的节点 |
| 起始 IP | IP 段的起始地址 |
| 结束 IP | IP 段的结束地址 |
| 网关 | 网关地址 |
| 子网掩码 | 子网掩码位数 |

创建后，系统会自动根据起始和结束 IP 生成该范围内的所有 IP 记录。

::: warning 注意
- 请确保您填写的 IP 段、网关和子网掩码与节点服务器上的实际网络配置一致，否则创建出来的实例将无法正常联网。
- IP 池创建后，起始和结束 IP 不可修改。如果需要扩展 IP 段，请创建新的 IP 池。
- 网关 IP 不要包含在 IP 段范围内，否则该 IP 会被分配给实例，导致整个网段不可用。
:::

## IP 分配 {#allocation}

- 创建实例时，系统自动从节点的 IP 池中分配一个空闲 IP
- 用户也可以购买额外 IP（如果套餐支持），额外 IP 同样从 IP 池中自动分配
- 实例删除后，IP 会自动回收到 IP 池中

::: danger 关于 IP 耗尽
当某个节点的所有 IP 池中没有空闲 IP 时，所有关联该节点的套餐都将无法创建新实例（即使套餐有库存）。系统会在创建实例时报错。

**建议**：定期关注各节点的 IP 使用情况。当空闲 IP 数量较低时，及时添加新的 IP 段或联系上游供应商获取更多 IP 资源。
:::

## 管理 IP {#manage}

在 IP 池详情页面，您可以查看每个 IP 的状态：

- **空闲**：可分配给新实例
- **已分配**：已关联到某个实例

您也可以手动将某个 IP 标记为保留状态，使其不参与自动分配。

::: tip
如果您发现某个 IP 存在被封禁或路由异常等问题，可以将其标记为保留状态，避免分配给新用户带来客诉。待问题解决后再取消保留。
:::
