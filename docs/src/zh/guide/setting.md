# 系统设置 {#setting}

管理面板的「系统设置」是集中管理 Novaix 各项运营配置的地方。SMTP 邮件、支付渠道、告警规则、注册开关等都在这里配置，无需修改配置文件和重启服务。

::: tip 系统设置 vs 配置文件
Novaix 有两个配置层级：`config.yaml` 负责底层基础设施配置（数据库、JWT、加密密钥、许可证等），修改后需要重启程序；管理后台的「系统设置」负责运营层面的配置（邮件、支付、告警等），修改后立即生效。
:::

## 站点设置 {#site}

| 设置项 | 说明 |
|--------|------|
| 站点名称 | 面板和邮件中显示的站点名称 |
| 站点 URL | 对外访问地址 |
| 站点 Logo | 面板中展示的 Logo |

## 注册设置 {#registration}

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| 开放注册 | 关闭 | 是否允许用户自助注册 |
| 邮箱验证 | 开启 | 注册时是否需要邮箱验证码 |

::: warning 注意
用户注册默认是关闭的。如果您希望用户可以自行注册，需要在此处手动开启。开启注册时强烈建议同时开启邮箱验证，并确保已配置好 SMTP 邮件服务。
:::

## 本地化设置 {#locale}

| 设置项 | 说明 |
|--------|------|
| 货币代码 | 订单和支付使用的货币（如 CNY、USD） |
| 时区 | 系统使用的时区 |

## 实例生命周期 {#lifecycle}

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| 暂停宽限天数 | 0 | 实例到期后多少天执行冻结（0 表示到期立即冻结） |
| 删除宽限天数 | 7 | 实例冻结后多少天自动删除 |
| 流量超限行为 | - | 流量超出限额时的处理方式 |

::: tip
建议合理设置宽限天数。`暂停宽限天数` 为 0 意味着实例一到期就会被冻结，用户来不及续费就会发现服务中断。可以适当设置 1-3 天的宽限期。`删除宽限天数` 建议至少保留 7 天，给用户足够的时间处理数据。
:::

## 安全设置 {#security}

| 设置项 | 说明 |
|--------|------|
| 最大登录尝试次数 | 连续失败多少次后锁定账号 |
| 会话超时时间 | 用户登录后多久需要重新认证（分钟） |
| 强制管理员双因素认证 | 是否强制管理员启用 TOTP |

## 邮件设置 {#mail}

SMTP 邮件服务配置，详见[邮件配置](./mail)。

## 支付设置 {#payment}

支付渠道配置，详见[支付配置](./payment)。每个支付渠道都有独立的启用开关。

## 告警设置 {#alert}

监控告警阈值配置，详见[监控与告警](./monitoring)。

## 维护模式 {#maintenance}

| 设置项 | 说明 |
|--------|------|
| 启用维护模式 | 开启后非管理员用户的所有请求都会返回 503 |
| 维护消息 | 维护期间展示给用户的提示信息 |

开启后管理员仍可正常访问管理面板，不受影响。适用于系统升级或维护期间临时关闭用户访问。

## 代理系统设置 {#agent}

代理商相关的配置，如佣金比例等。详见[代理系统](./agent)。

## 其他设置 {#other}

- **自动备份**：实例自动备份相关配置
- **反向 DNS**：rDNS 相关配置
- **邮件入站**：邮件回复工单的 Webhook 配置
- **法律文档**：服务条款和隐私政策内容
- **首页内容**：用户面板首页展示内容

## 操作日志 {#logs}

系统会自动记录管理员的所有变更操作（创建、修改、删除等）。您可以在「操作日志」中查看完整的审计记录，支持导出到 Excel。

## 数据导出 {#export}

管理面板支持将以下数据导出为 Excel 文件：

- 用户列表
- 订单列表
- 支付记录
- 操作日志
